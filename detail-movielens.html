Project Overview

In the digital era, the explosion of big data and digital information has become an inevitable phenomenon. With entertainment industry services like Netflix and Amazon Prime offering thousands of movies, users face the challenge of finding content that matches their preferences. This project implements a machine learning-based movie recommendation system to solve this problem.

Business Understanding

Without recommendation systems, entertainment streaming businesses would experience decreased user engagement, ultimately impacting revenue. Recommendation systems are crucial for:

  • Personalization: Providing recommendations based on individual user preferences
  • Overcoming Human Limitations: Using AI to efficiently process big data
  • Optimizing User Experience: Helping users find content matching their interests, reducing churn
  • Leveraging User Data: Continuously improving accuracy through data analysis
  • Enhancing Marketing: Promoting less popular content to relevant users

Problem Statements

  • How to build a movie recommendation system that provides relevant and personalized recommendations?
  • What dataset is needed to build an effective recommendation system?
  • How to evaluate the performance of the built recommendation system?

Goals

  • Build a movie recommendation system with error below 0.2
  • Identify and collect necessary datasets
  • Evaluate the performance of the built system

Solution Approach

Two methods are implemented:

  • Content-Based Filtering: Using cosine similarity based on genre and tags
  • Neural Network: Learning patterns from user preferences and ratings

Dataset

The project uses the MovieLens Latest dataset containing:

  • 33,832,162 ratings
  • 2,328,315 tags
  • 86,537 movies

Data Files

  • ratings.csv: userId, movieId, rating (0.5-5.0), timestamp
  • tags.csv: userId, movieId, tag, timestamp
  • movies.csv: movieId, title, genres
  • links.csv: movieId, imdbId, tmdbId

Data Preparation

1. Downsampling

Performed to balance rating distribution and reduce training time, making the dataset more manageable while maintaining representativeness.

2. Data Merging

  • Movies + Tags: Combined to get complete film descriptions based on genre and user-given tags
  • Movies + Ratings: Merged to obtain complete rating information for each film

3. Data Transformation

  • User Preferences: Created average rating data for each genre per user
  • Item Data: Encoded genres as binary columns (1 = has genre, 0 = doesn't have genre)
  • Rating Data: Extracted as target variable

4. Scaling & Splitting

  • User and item data scaled with StandardScaler (positive = liked, negative = disliked)
  • Rating data scaled with MinMaxScaler (0-1 range)
  • Split: 80% training, 10% validation, 10% testing

Modeling

1. Content-Based Filtering with Cosine Similarity

This method calculates similarity between films based on genres and tags using CountVectorizer for feature extraction. The system returns top 10 similar movies based on cosine similarity scores.

2. Neural Network Approach

Uses separate sequential models for user preferences and item features, combined via dot product. The architecture includes:

User Model
  • • Input layer (user preferences)
  • • Dense layers with ReLU
  • • Dropout (0.2) for regularization
  • • Normalization layer
Item Model
  • • Input layer (movie features)
  • • Dense layers with ReLU
  • • Dropout (0.2) for regularization
  • • Normalization layer
Best Architecture Found (via GSD)
  • • Layer 1: 512 neurons (ReLU, Dropout 0.2)
  • • Layer 2: 256 neurons (ReLU, Dropout 0.2)
  • • Layer 3: 64 neurons (Linear activation)
  • • Optimizer: Adam (lr=0.001)
  • • Training: 100 epochs, batch size 256

Key Insights

  • Content-based filtering excels at recommending similar content but ignores user rating patterns
  • Neural networks can predict personalized recommendations for new users
  • Combining both approaches provides comprehensive recommendation capabilities
  • Left-skewed rating distribution (3.0-4.0) required careful preprocessing
  • Model achieved goal of < 0.2 error with actual MSE of 0.06
